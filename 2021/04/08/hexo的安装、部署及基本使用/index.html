hexo的安装、部署及基本使用

发表于 2021-04-08 更新于 2024-01-12 分类于 hexo
hexo环境部署、初始化到让别人能看到你的blog并发布

什么是hexo

hexo是一个基于node.js的静态博客，作者是个台湾人，对中文使用人很友好，官方的文档非常全面，更新勤快！夸一口。

你需要啥子准备

  • 一个Github账号

hexo框架是放在Github上的，一个GitHub账号可以方便的扒拉hexo框架和其他学习资料下来。

记得取一个你喜欢的账户名，若是放在GitHub上托管，那么你的博客名就是你的用户名，但是也不必担心，可以改。

https://github.com/

  • git

代码版本管理器，一开始分不清楚git和GitHub之间的关系，git帮助你回退到任何一个项目的历史版本，而GitHub中放的是你的项目代码。

换个话说，git可以配合别的代码管理器使用，比如gitee。

GitHub看成是一个只放代码相关的网盘

在git的使用过程中，配合git的官方文档食用更佳，官方有中文文档。

一定要记得！！！下载最新的版本，否则你在部署到GitHub上会出错，叫logon faild，更新一下就好了

$git update-git-for-windows

http://git-scm.com/

  • node.js

直接在官网下载就好了，hexo主要是基于node.js的静态网页来搭建的，node.js继承了npm指令，一些hexo的组件，需要通过npm指令来下载。

http://nodejs.cn/

环境配置

  • 先查看一下版本

打开cmd

node -v
npm -v
npm config set prefix “F:\nodejs\node_global”   //配置全局模块存放路径
npm config set cache “F:\nodejs\node_cache”   //cache路径
  • 环境变量

需要在nodejs安装路径下添加node_cache、node_global—-node_modules，由npm加载的文件将放到node_modules中。

用户变量：Path中添加~~\nodejs\node_global

系统变量：新建NODE_PATH 值为~~\nodejs\node_global\node_modules

系统变量：Path中添加nodejs的路径、node_cache的路径、Git安装路径下的Git\cmd（git安装时勾选path将自行创建）

Hexo初始化

在你希望放置blog的位置下右键打开git bash

hexo--cmd--npm install -g hexo -cli
hexo init <你的blog文件夹名>
cd <你的blog文件夹名>
npm install

然后你就会发现在blog文件夹下已经自动在本地组件好了hexo架构

Hexo操作和部署

  • hexo g ———把本地的代码部署到本地服务器
  • hexo s ———打开本地服务器
  • hexo d ———将本地代码部署到GitHub上

要想部署到GitHub上，你需要在自己的GitHub上创建你的代码仓库，以自己的用户名为仓库名，部署后可以以仓库名.github.io拜访blog。

在部署之前，需要设置本地的git全局用户名和邮箱

git config --global user.name "你的用户名"
git config --global user.email "你的邮箱"

全都部署好后执行hexo d，如果出错代码为443

git config --global --unset http.proxy

个性化设置

打开hexo的官网下载大佬们布置好的theme，还有添加一些字数统计、评论、代码高亮等等的设置和插件，详情请google和查看官网
https://hexo.io/

Hexo写作

是基于Markdown的语法来书写并被JS渲染，所以熟悉基本的Markdown语法是必要的，在使用Markdown编辑器的时候，可以使用编辑器自带的编辑器快速布局，用来生成H1、2、3……的标题，以及无序、有序编号，代码块的引用等。

在自己的Hexo项目文件结构中，scaffolds文件夹下，有三种默认的模板（draft、post、page-不常用），在bash中使用

hexo new [draft/post/ ] ["title"]

如果有一些主题没有写好，可以先用draft来创建一个草稿，当文章编辑好之后，再使用

hexo publish [draft] ["title"]

来将draft放到post中，等待发布
