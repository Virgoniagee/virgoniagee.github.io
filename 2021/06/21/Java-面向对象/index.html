类的成分、三大特性、关键字

类及类的成员

属性

属性vs局部变量

  • 相同点：

    1. 定义格式
    2. 先声明再使用
    3. 作用域内生效

  • 不同点：

    1. 声明位置：

      属性：class{ }内
      局部变量：方法内、方法形参、代码块内、构造器形参、构造器内部

    2. 修饰权限：

      属性：声明时指明权限，常用的权限修饰符：private、public、缺省、protected ->封装性
      局部变量：不能用

    3. 初始化：

      属性：定义后自动初始化为类型默认值
      局部变量：需要声明，使用必须显示赋值，手动的

    4. 内存加载位置：

      属性：非static在堆空间
      局部变量：在栈空间

赋值顺序

  1. 默认初始化
  2. 显示初始化/代码块中赋值(看先后顺序)
  3. 构造器中初始化
  4. 通过”对象.方法” 或 “对象.属性”的方式

方法

方法结构

修饰符 返回值 方法名(形参){
方法体;
//return 返回值}

方法重载

  • 在同一个类中，允许存在一个以上的同名方法，只要它们的参数个数或者参数类型不同即可
  • 两同一不同：同类、方法名****不同参数列表、个数、类型

形参

在方法结构中作为参数传入的局部变量，因为是由“外部”传入的参数，不需要显式声明，但是作用于方法内部，是否会对原本传入的参数有所改变要看什么类型，分析其内存位置。

Java的形参是值传递，但是不分为基本数据类型和引用数据类型，String是例外。

值传递

Java中参数传递机制：值传递

  • 规则：

如果变量是基本数据类型，此时赋值的是变量所保存的数据值。

如果变量是引用数据类型，此时赋值的是变量所保存的数据的地址值。

  • 两个概念：

形参：方法定义时，声明的小括号内的参数

实参：方法调用时，实际传递给形参的数据

形参空有形没有值，其值有传入的实参决定

递归

递归就是一个方法调用它自己

  • 递归的必须项：
    可以拆解为除了数据规模以外，求解思路完全相同的子问题。
    存在终止条件

  • 含义：
    递归问题必须可以划分为若干小规模问题，并与原问题形式有相同子问题，解题思路相同。

    是对原问题从大到小的拆解，并且是一个单向有明确终点的拆解，并且有一个临界点，当达到临界点后，往回追朔解决。

构造器

Constructor

  • 作用：
    1. 创建对象
    2. 初始化对象的信息

  • 使用：

    1. 如果没显式的定义类的构造器的话，则系统默认提供一个空参的构造器，一旦显示定义了非空构造器，则空构造器需要自己定义
    2. 定义构造器的格式：权限修饰符 类名(形参列表){}
    3. 一个类中定义的多个构造器，彼此构成重载
    4. 一个类中，至少会有一个构造器。

JavaBean

  • 类是公共的，public class
  • 一个无参的公共的构造器
  • 定义属性并有对应的get、set方法

代码块

  • 把一些不能在class中直接写入的代码包住
  • 常用作一些方法形参的提前初始化

内部类

一个类A声明在另一个类B中，则类A就是内部类，类B称为外部类

Object

java.lang.Object

作为所有class的父类

.equals()

boolean (Object obj)

比较的是堆中的值，被String重写，常被继承的子类重写。

.getClass()

返回运行时类，常用于泛型和反射

.hashCode()

返回当前对象的哈希值，冲突率极低，基于存储地址和类的特性来确定的。

至于为什么是31，不是1不是2也不是100，是因为31是2^5-1，可以很快的用位运算符得到，至于为什么不是别的2的倍数-1，是因为比31小的冲突多，比31大的和31性能差不多，但是占用了更多的内存

.toString()

返回类的信息(String)，若是没有重写过，则返回字符串表示形式，重写后常作为当前类的信息。

三大特性

封装

  • 隐藏对象内部的复杂性，只对外公开简单的接口。便于外界调用，从而提高系统的可扩展性、可维护性。通俗的说，把该隐藏的隐藏起来，该暴露的暴露出来。这就是封装性的设计思想。
  • 代码体现：
    1. 属性私有化，提供公共的方法来get、set
    2. 不对外暴露私有的方法
    3. 单例模式(将构造器私有化)
    4. 修饰类为缺省，让其不能在其他包内调用

4种权限都可以用来修饰类的内部结构：属性、方法、构造器、内部类
修饰类的话，只能使用：缺省、public

继承

继承是分类的艺术，比如生物种属的划分更为贴切。

越高的种属拥有越概括性的属性和特点，子子孙孙的继承拓展出自己的特性。

格式

class A extends B{ }

A:子类、派生类、subclass

B:父类、超类、基类、superclass

特点

  • 减少了代码的冗余，提高了代码的复用性
  • 便于功能的扩展
  • 为之后多态性的使用，提供了前提

    subclass

  • 继承父类后，得到父类所有的属性和方法，父类中private修饰的属性和方法不能直接调用，可以用get/set或使用父类中public方法中被调用
    子类可以拓展功能，或重写方法

    说明

  • 一个类可以被多个子类继承。
  • Java中类的单继承性：一个类只能有一个父类
  • 子父类是相对的概念。
  • 子类直接继承的父类，称为：直接父类。间接继承的父类称为：间接父类
  • 子类继承父类以后，就获取了直接父类以及所间接父类中声明的属性和方法

    重写

  • 子类重写的方法的方法名和形参列表与父类被重写的方法的方法名和形参列表相同
  • 子类重写的方法的权限修饰符不小于父类被重写的方法的权限修饰符

返回值：
父子皆空、臭味相投、为父则刚（extends或throws）

多态

理解为一个事物的多种形态。

父类的引用指向子类的对象（或子类的对象赋给父类的引用）

假设声明一个class A 其父类为Object

Object a = new A( );

向上转型才是多态

抽象类、接口的使用肯定体现了多态性。

虚拟调用

只适用于方法，不适用于属性

有了对象的多态性以后，我们在编译期，只能调用父类中声明的方法，但在运行期，我们实际执行的是子类重写父类的方法。

编译，看左边；运行，看右边。

前提

类的继承关系
方法的重写

非多态 向下转型

  • 使用强制类型转换符( )
  • 使用强转后，编译时父类转化为实际的子类，使子类可以调用其特有的方法
  • 使用 a instanceof A 判断a是否为A的实例，表示a是A类或A的父类，极大的避免了ClassCastException的异常

关键字

static

修饰类的内部结构

属性、方法、代码块、内部类

属性

名为静态变量，表示，尽管new了很多同类对象，但是静态变量是共享的

说明

  • 静态变量随着类的加载而加载。可以通过”类.静态变量”的方式进行调用

  • 静态变量的加载要早于对象的创建。

  • 由于类只会加载一次，则静态变量在内存中也只会存在一份：存在方法区的静态域中。

    方法

  • 随着类的加载而加载，可以通过”类.静态方法”的方式进行调用

  • 静态方法中，只能调用静态的方法或属性

  • 非静态方法中，既可以调用非静态的方法或属性，也可以调用静态的方法或属性

    代码块

    要用修饰符只能用static

  • 内部可以输出语句

  • 随着类的加载而执行,而且只执行一次

  • 作用：初始化类的信息

  • 如果一个类中定义了多个静态代码块，则按照声明的先后顺序执行

  • 静态代码块的执行要优先于非静态代码块的执行

  • 静态代码块内只能调用静态的属性、静态的方法，不能调用非静态的结构

由父及子，静态先行。

内部类

  • 创建静态内部类
    public class out{
        static class in{
        }
    }
    out.in clazz = new out.int();
  • 创建非静态内部类
    public class out{
        class in{
        }
    }
    out.in clazz = new int();

final

可以用来修饰：类、方法、变量

类

此类不能被其他类所继承。

String类、System类、StringBuffer类

方法

表明此方法不可以被重写

变量

修饰为final的变量 可以视为一个常量

常用大写字母表示

声明时没有初始化，则在第一次赋值时表现为常量

static final 用来修饰属性：全局常量

abstract

修饰类、方法

abstract不能用来修饰私方法、静态方法、final的方法、final的类

类

抽象类

  • 此类不能实例化

  • 抽象类中一定有构造器，便于子类实例化时调用（涉及：子类对象实例化的全过程）

  • 开发中，都会提供抽象类的子类，让子类对象实例化，完成相关的操作

    方法

    抽象方法

  • 抽象方法只方法的声明，没方法体

  • 包含抽象方法的类，一定是一个抽象类。反之，抽象类中可以没有抽象方法的。

  • 若子类重写了父类中的所的抽象方法后，此子类方可实例化

  • 若子类没重写父类中的所的抽象方法，则此子类也是一个抽象类，需要使- 用abstract修饰

    interface

    Java中只能单继承，接口可以多实例。

接口和类是并列的两个结构

return

使用在方法体中

用于结束方法，带返回值类型的方法时用于返回数据

return后不能声明语句

package/import

更好的实现项目管理

MVC

  • M model 主要处理数据
    bean、dao、db

  • V view 显示数据
    utils、ui

  • C controller 处理业务逻辑
    service、base、adapter、activity

    JDK

  • java.lang | 该包提供了Java编程的核心基础类，例如 Object、Math、String、StringBuffer、System、Thread等，不使用该包就很难编写Java代码了

  • java.awt | 提供了创建界面和绘制图形图像的所有类，构成抽象窗口工具、GUI设计

  • java.net | 该包提供实现网络应用与开发的类。

  • java.io | 该包通过文件系统、数据流和序列化提供系统的输入与输出。

  • java.util | 包含一些使用工具类，定义系统特性、使用日期日历

  • java.text | Java格式化的类

  • java.sql | 包含JDBC数据库编程的相关类和接口

    package

    实现项目中类的管理

  • 使用package声明类或接口所属的包，声明在源文件的首行

  • 包，属于标识符，遵循标识符的命名规则、规范(xxxyyyzzz)、“见名知意”

  • 每”.”一次，就代表一层文件目录。

    import

  • 在源文件中显式的使用import结构导入指定包下的类、接口

  • 声明在包的声明和类的声明之间

  • 如果需要导入多个结构，则并列写出即可

  • 可以使用”xxx.*”的方式，表示可以导入xxx包下的所结构

  • 如果使用的类或接口是java.lang包下定义的，则可以省略import结构

  • 如果使用的类或接口是本包下定义的，则可以省略import结构

  • 如果在源文件中，使用了不同包下的同名的类，则必须至少一个类需要以全类名的方式显示。

  • 使用”xxx.*”方式表明可以调用xxx包下的所结构。但是如果使用的是xxx子包下的结构，则仍需要显式导入

  • import static:导入指定类或接口中的静态结构:属性或方法。
