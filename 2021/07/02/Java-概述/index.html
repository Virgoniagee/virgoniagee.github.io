JAVA特点

《Java核心技术》卷Ⅰ 1.2

  • 简单性

  • 面向对象

    • 将程序设计重心放在数据对象和数据对象的接口上
    • 封装、继承、多态

  • 分布式

    • 方便处理应用层协议，像访问本地文件一样用URL访问网络

  • 健壮性

    • 编译器能检测出其他语言在运行时出现的逻辑错误
    • 不需要维护指针

  • 安全性

    • 可以构建防病毒、防篡改的系统，沙箱系统
    • 防止堆栈溢出：防病毒和蠕虫攻击
    • 不破坏进程空间外内存
    • 未经授权不可读写文件

  • 体系结构中立

  • 可移植性

    • JVM使得同一个.class文件可以在不同系统下得到相同结果

  • 解释型

  • 高性能

  • 多线程

  • 动态性

环境配置

  • 目的为让cmd命令在任何位置都能运行执行文件

  • jdk1.8之后就不用配置classpath的路径了

  • https://www.runoob.com/java/java-environment-setup.html

  • JAVA_HOME        *****\jdk1.8.0_91 
CLASSPATH        .;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar;   
Path             %JAVA_HOME%\bin;%JAVA_HOME%\jre\bin;

  • 若安装了两个版本的jdk，导致不能运行java.exe，即javac和java版本不相符时

    • 删除一个版本
    • 确定环境变量，java的版本默认为最新安装的版本，javac的版本需要到系统环境变量中修改，直接修改JAVA_HOME为最新安装的jdk路径，若是还未成功，在path中将关于java的环境变量提前上移，玄学为从上往下找，找到就不找了。
    • 此玄学的关键在于，%JAVA_HOME%\bin;%JAVA_HOME%\jre\bin; 置于C:\Program Files\Common Files\Oracle\Java\javapath;C:\Program Files (x86)\Common Files\Oracle\Java\javapath;上方

  • 配置好环境后在cmd中输入 java -version 查看版本、 java 查看指令

JDK JRE JVM

  • jdk>jre>jvm
  • jvm是java虚拟机
  • jre是程序运行环境 jre=jvm+java核心类库
  • jdk是开发人员程序开发包 jdk=jre+java开发工具（bin\*.exe）

注释

  • 单行注释// 描述一些变量或方法
  • 多行注释/* */ 常注释一段代码用于调试
  • 文档注释/** */ 对方法的描述，生成API文档

Java

Java无疑是强大的，由于JVM的存在，使得其可以跨平台运行

Java俗称万事万物皆对象，通过封装对外仅暴露API，用一些public的方法实现对属性和方法的调用

与数据库交互的时候，更是将每个库的表分门别类的当作对象，放在Bean中，实现数据库中每一项记录对应一个Java对象

JDK的迭代让Java能快速跟进时代，适应新的开发需求，同时自己也可以开发自己的类库来实现定制化的功能
